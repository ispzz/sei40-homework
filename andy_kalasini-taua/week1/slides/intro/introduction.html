Traits of a Good SEi Student

  • Writes as much actual code as they can, checks newly-written code at each step by actually running it
  • Fearless about trying out new things for themselves, natural curiosity about code tricks and tools ("I wonder if I can do X in this language?"; "Would Y work?")
  • Shameless about asking for help when they get stuck, early and often
  • Relentless about doing their own research and finding their own answers
  • Selfless in helping other students fix their bugs
  • Stays positive about their progress in the course; avoids comparisons to more advanced students; instead, compares to themselves from a week ago
  • Does as much homework as they can; starts homework early in the night, so they have enough time to get help with any issues
  • Challenges themselves by doing extra in homework & projects

B-b-but What IS  Programming?

  • A list of painfully literal, clear instructions to a dumb computer to get it to do something useful for you
  • A form of writing / communication
    • Hence the need for clarity (over cleverness)
    • Code style/layout: optimise for readability, communication of intent
  • Giving clear names to things: labelling, abstracting, grouping, collecting... pattern recognition
  • A way of thinking: solving problems in a disciplined, highly structured (logical, hierarchical) style
  • It will take some time for this new way of thinking to click - but it's a nice feeling when it does... and then "it sort of improves itself while you‚Äôre not doing anything"

In this course, the transferable core skills of programming itself are more important than the details of any particular language, framework or tool
